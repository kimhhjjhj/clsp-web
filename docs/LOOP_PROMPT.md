# CLSP 개선 루프 프롬프트

> 이 프롬프트 본문을 `/loop` 명령의 인자로 사용합니다.
> 루프가 매 사이클마다 아래 지시를 실행합니다.
> 전체 배경은 `AGENTS.md`의 `engineer-mindset` 섹션에서 이미 적용됩니다.

---

## 실행 방법

### 수동 (현재 세션에서 반복 트리거)
사용자가 **"다음 사이클"** 또는 **"계속"** 이라고 지시할 때마다, 아래 루프 본문을 1회 실행.

### 자동 (`/loop` 슬래시 커맨드 사용)
```
/loop
```
입력 후 아래 "루프 본문"을 그대로 프롬프트로 사용. Claude가 대기 시간을 자율 판단하며 반복 실행한다.

### 중단 조건
- Tier 0~3 항목이 모두 `[x]`로 완료됨
- 사용자가 새로운 방향성 지시
- 도메인·비즈니스 판단이 필요한 블로커 (사용자 응답 대기)

## 루프 본문 (복붙용)

```
CLSP 웹사이트 개선 사이클을 1회 실행한다.

## 스텝

### 1. 상태 점검
- `docs/IMPROVEMENT_BACKLOG.md` 를 읽고 `[ ]` 상태의 최상위 우선순위 항목 1개 선택
  · Tier 낮은(숫자 작은) 것부터, 같은 Tier 내에서는 완료 조건이 명확하고 독립적인 것 먼저
  · 이미 진행 중인 다른 항목과 충돌 없는지 확인 (예: 인증 도입 중이면 권한 작업 병행 금지)
- 선택한 항목을 `[~]` 로 표시 (진행 중)

### 2. 계획 수립
선택한 항목에 대해:
- 필요한 스키마 변경 / API / 컴포넌트 / 스크립트를 식별
- 완료 조건을 구체 수치·체크리스트로 재정의
- 도메인 의미론적 판단이 필요하면 즉시 사용자에게 질문 후 대기

### 3. 구현
AGENTS.md의 engineer-mindset 원칙 준수:
- 만든 기능은 직접 써본다 (curl·스크립트·페이지 HTTP)
- 파괴적 변경은 비파괴 우회 (기존 DB 값 보존, 새 컬럼은 nullable)
- 작업 단위는 작게, 의미 있는 지점마다 커밋
- 모호할 땐 자동 적용 대신 사용자 확인 뱃지

### 4. 검증
- `npx tsc --noEmit` 통과 (관련 파일만 grep 해서 에러 없음 확인)
- dev 서버가 켜져 있다면 관련 페이지 200 응답 확인
  · 목록 페이지 / 상세 페이지 / API 엔드포인트
- 필요시 `scripts/` 밑에 검증용 mjs 스크립트로 실데이터 검증

### 5. 커밋·푸시
- 변경 파일 명시적으로 `git add`
- 커밋 메시지 규칙:
  · `feat:` / `fix:` / `refactor:` / `chore:` 접두사
  · 제목 1줄 (한국어, 72자 이내)
  · 본문: 변경 이유, 구현 요약, 완료한 백로그 항목 번호·제목 참조
  · 마지막 줄: `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>`
- `git push` 까지 자동

### 6. 백로그 갱신
- 선택한 항목을 `[x]` 로 체크
- 구현하며 새로 발견한 개선 아이디어가 있으면 적절한 Tier에 `[ ]` 로 추가
- 블로커(사용자 판단 필요)로 중단된 경우 `[?]` 표시 + 블로커 내용 한 줄 메모

### 7. 보고
사용자에게 **2~4줄 요약**만 보고:
1. 완료한 항목 제목 + 커밋 해시
2. 주요 구현 포인트 1~2개
3. 다음 사이클 후보 항목 (자동 선택 예정)
4. (있을 때만) 사용자 판단 필요한 사항

## 원칙

- 한 사이클 = 한 항목. 욕심 내지 않는다.
- 사용자는 대화에 없을 수 있다. 자율적으로 끝까지 처리하되, 도메인 판단 필요 시 즉시 `[?]`로 표시하고 보고 후 중단.
- 백로그 순서가 어색해지면 상위 Tier 재정렬 허용 (단, 재정렬 이유를 커밋 메시지에 남길 것).
- 각 사이클 종료 시 반드시 **커밋·푸시까지 완료**해야 한다. 롤백 가능하게 작은 단위로.

## 완료 조건

- Tier 0~3의 모든 `[ ]` 가 `[x]` 또는 `[?]` 가 된 상태
- 이때는 "Tier 0~3 완료. Tier 4 진입 승인 필요" 라고 보고 후 루프 자발 종료.
```

## 사용 팁

### 어느 시점에 수동으로 탈출하고 싶다면
- 루프 중 ESC 또는 `/stop`
- 다음 대화에서 "루프 중단하고 [특정 작업]" 이라고 지시하면 루프 무시하고 해당 작업 우선

### 블로커(`[?]`)가 여러 개 쌓이면
- 사용자가 일괄 검토해 `[ ]` 로 되돌리거나 Tier 재조정
- 막힌 항목만 스킵하려면 `[~~]` 또는 `~~` 취소선 처리 후 하단 "보류" 섹션에 이관

### 새 PRD/TRD 요구사항 추가 시
1. 백로그 해당 Tier에 `[ ]` 항목 추가 (완료 조건 명시 필수)
2. 루프는 자동 감지해 다음 사이클에 반영
